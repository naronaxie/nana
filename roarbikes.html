About the project

This case study is an overview of the website design for a fictional company called Roar Bikes. Roar Bikes is a manufacturer of small run, contemporary bicycles sold exclusively from their own website. Roar Bikes currently have 3 models of bike to purchase (Siamese, Sphynx, Bengal). The goal of the project was to create an e-commerce website for customers to browse and purchase.

Role

User Interface and User Experience Designer

User Research, Information Architecture, Prototyping, Testing, Visual Design

Deliverables

  • Wireframes for client approval.
  • High fidelity prototype.
  • User Testing - Hallway testing & unmoderated remote testing.
  • UI Assets for developers.

Empathizing with cyclists

I conducted research to find out what the most common problems are for bikers are. My conclusions are as follows:

  • Individuals felt frustrated that the one-size-fits-all bicycles do not fit all body types and sizes.
  • Individuals want control over the aesthetic of their bicycles.
  • Individuals want practical but unique bicycles.
  • Some individuals want features that are not included in a standard model.

User Persona

A basic user persona was given to me, however I wanted to focus more on his personality and frustrations based on my research about cyclists. By understanding the user more, I was able to focus the designs based on his needs.

Information Architecture

Based on the user persona and client deliverables, I created a sitemap. I wanted to create an experience that allows the user to access the bicycle styles from both the home page and the individual pages. I placed focus on simplicity, interconnectivity and accessibility of the site.

Sketching Ideas

I began the design process by sketching ideas of how I wanted the layout to be. Because the company has a strong emphasis on quality, I designed the product page to highlight the specifications of the bicycle parts. In the early stages I wanted to create a highly customizable, interactive 3D version of the bicycle that would update in real time in response to user changes. Because it is a small, local company, Roar Bikes requested that social media be included on most pages to highlight owner photos.

Wireframing

During wire framing, I focused on interconnectivity and the sitemap throughout the design. I spent time prototyping the interaction between web pages so that the bicycle would roll in from the right side of the page into the center whenever the user changed colors. I also designed it so that the colors changed as you scrolled, and it would only go down when you were at the last color.

One of the largest challenges that I faced was the scope for one of my ideas was too large. My initial idea for an interactive and customizable 3D model coded in WebGL was impractical for everyday use. It would slow down the webpage and therefore result in less user retention rates. In my final prototype I decided to keep it simple instead by having the bicycle auto animate to the next color while staying still.

I also changed my original design so that the user can scroll down freely without being stuck within the colors area. The main concern with the first idea was that the user would get tired of scrolling if there were too many color options, especially if they just want to scroll to the features.

Visual Design

At this point I created the visual design for the interface of the website. I started creating the design based on the original logo, but when the logo size was reduced it was not legible. I adopted the original lion design, but chose a bolder font with a bright yellow color. I built the color palette around that while emphasizing bold colors. Because this is a fictional company, bicycle assets were borrowed from Mission Bicycle.
